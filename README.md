# Gonka Telegram Broadcaster

Служебный бот [@gonkasupport_bot](https://t.me/gonkasupport_bot) для отправки одного сообщения в заранее зарегистрированные Telegram-группы и темы.

## Возможности первой версии

- доступ только для числовых Telegram ID из `ADMIN_USER_IDS`;
- регистрация группы или темы командой внутри неё;
- рассылка во все либо выбранные направления;
- текст, фото, видео, документы и другие обычные сообщения через Telegram `copyMessage`;
- обычная или тихая отправка;
- обязательный предпросмотр и кнопка подтверждения;
- защита от повторного нажатия;
- пауза между сообщениями и обработка Telegram `retry_after`;
- продолжение работы при ошибке отдельной группы;
- аварийная остановка после пяти последовательных ошибок;
- SQLite-журнал кампаний и доставок;
- автоматическое отключение групп, из которых удалили бота;
- отсутствие сторонних Python-зависимостей.

Бот публикует сообщения от своего имени. Он не рассылает личные сообщения пользователям и не имитирует аккаунт сотрудника.

## Требования

- Python 3.11 или новее либо Docker;
- постоянно работающий сервер с доступом к `api.telegram.org`;
- токен [@gonkasupport_bot](https://t.me/gonkasupport_bot);
- числовой Telegram user ID хотя бы одного оператора.

GitHub Actions не используется для работы бота: long polling должен быть запущен постоянно.

## 1. Настройки BotFather

Для `@gonkasupport_bot`:

1. `Allow Groups` — включено.
2. Group Privacy — включено. Боту не нужно читать обычную переписку.
3. Токен хранится только в секрете сервера.

Команды для `/setcommands`:

```text
start - открыть меню
help - показать справку
broadcast - создать рассылку
broadcast_silent - создать тихую рассылку
groups - показать зарегистрированные группы
history - показать последние рассылки
whoami - показать Telegram user ID
register - зарегистрировать текущую группу или тему
unregister - отключить текущую группу или тему
cancel - отменить текущий черновик
```

## 2. Первый запуск для определения user ID

Настоящий токен никогда не добавляется в Git.

Создайте локальный `.env` из `.env.example` и заполните только токен:

```dotenv
TELEGRAM_BOT_TOKEN=полученный_в_BotFather_токен
ADMIN_USER_IDS=
DATABASE_PATH=./data/broadcaster.sqlite3
```

Запустите бот и в личном чате отправьте `/whoami`. Эта команда безопасно показывает отправителю только его собственный числовой ID и работает до настройки администраторов.

Остановите бот, внесите ID в `.env` и запустите снова:

```dotenv
ADMIN_USER_IDS=123456789
```

Пока `ADMIN_USER_IDS` пуст, ни одна привилегированная команда не выполняется.

## 3. Запуск без Docker

Из корня проекта:

```bash
python3 -m broadcaster
```

Библиотеки устанавливать не требуется.

## 4. Запуск через Docker Compose

Создайте `.env`, затем:

```bash
docker compose up -d --build
docker compose logs -f broadcaster
```

SQLite хранится в именованном Docker volume `broadcaster-data` и не исчезает при пересоздании контейнера.

## 5. Регистрация группы

1. Добавьте `@gonkasupport_bot` в согласованную группу.
2. Убедитесь, что бот может отправлять сообщения.
3. Оператор из `ADMIN_USER_IDS`, являющийся администратором группы, пишет:

```text
/register validator-ru
```

Alias должен содержать 2–40 латинских букв, цифр, `_` или `-`.

Для группы с Topics выполните команду внутри нужной темы. Бот сохранит `message_thread_id`, и рассылки будут попадать именно туда.

Посмотреть результат в личном чате:

```text
/groups
```

Чтобы отключить направление, выполните `/unregister` в той же группе или теме.

## 6. Создание рассылки

Во все активные группы:

```text
/broadcast
```

Только в выбранные группы:

```text
/broadcast validator-ru gw2-support host-17
```

Без звукового уведомления:

```text
/broadcast_silent
```

После команды:

1. Бот показывает получателей.
2. Отправьте ему одно готовое сообщение.
3. Бот копирует сообщение обратно как предпросмотр.
4. Проверьте текст, ссылки, вложение и количество групп.
5. Нажмите `Разослать` или `Отмена`.
6. Дождитесь итогового отчёта.

Пересланное из защищённого чата сообщение Telegram может запретить копировать. В таком случае создайте новое сообщение непосредственно в личном чате с ботом.

Черновик действует 15 минут. Новая команда `/broadcast` отменяет предыдущий незавершённый черновик этого оператора.

## 7. Переменные окружения

| Переменная | Обязательная | Значение |
|---|---:|---|
| `TELEGRAM_BOT_TOKEN` | да | секретный токен BotFather |
| `ADMIN_USER_IDS` | после bootstrap | числовые ID через запятую |
| `DATABASE_PATH` | нет | `./data/broadcaster.sqlite3` |
| `LOG_LEVEL` | нет | `INFO` |
| `POLL_TIMEOUT_SECONDS` | нет | `30` |
| `SEND_DELAY_SECONDS` | нет | `0.2`, примерно пять отправок в секунду |
| `DRAFT_TTL_MINUTES` | нет | `15` |

## 8. Безопасность

- Не присылайте токен в чат и не сохраняйте его в GitHub-файлах.
- Настоящий `.env` исключён через `.gitignore`.
- Username не используется для авторизации — только числовой user ID.
- Каждая команда и callback-кнопка повторно проверяют оператора.
- `/register` требует, чтобы оператор также был администратором целевой группы.
- Подтверждение кампании атомарно: повторное нажатие не запускает вторую очередь.
- После аварийного перезапуска незавершённая отправка помечается `interrupted` и не возобновляется автоматически, чтобы не создавать дубли.

Если токен раскрыт, немедленно перевыпустите его в BotFather и замените секрет на сервере.

## 9. Тесты

```bash
python3 -m unittest discover -s tests -v
```

Тесты используют поддельный Telegram API и не отправляют реальные сообщения.

## 10. Резервное копирование

Рабочие данные находятся в SQLite по `DATABASE_PATH`. Резервную копию нужно делать на уровне сервера или Docker volume.

Не копируйте открытый файл простым побайтовым копированием во время активной записи. Используйте штатный SQLite backup либо кратковременно остановите сервис перед резервным копированием.

## Ограничения первой версии

- одна Telegram-публикация за кампанию;
- нет альбомов и серии сообщений;
- нет расписания;
- нет веб-интерфейса;
- нет автоматического редактирования уже разосланных сообщений;
- retry only failed destinations будет добавлен после проверки основного сценария.
